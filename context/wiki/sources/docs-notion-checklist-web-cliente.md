# Checklist de entrega de web de cliente

## Resumen

Checklist operativo para revisar una web de cliente antes de entregarla. Cubre preparacion, contenido, SEO, formularios, responsive, deploy, Supabase y cierre documental.

## Bloques de revision

- Antes de construir: objetivo, cliente, dominio, servicios, contacto, imagenes, referencias y stack.
- Contenido: hero, servicios, CTAs, contacto, legales y textos no genericos.
- SEO: title, description, canonical, Open Graph, imagen OG, schema, sitemap, robots y alt text.
- Formularios: validacion, envio real, estados de exito/error, almacenamiento o email y privacidad.
- Responsive y UX: mobile, tablet, desktop, menu movil, CTAs visibles y ausencia de solapes.
- Deploy: build, configuracion, variables, dominio, HTTPS, redirecciones y cache.
- Supabase: pooler en produccion, RLS, claves privadas y pruebas reales.
- Cierre: documentacion, repo, decisiones tecnicas e instrucciones para cliente.
