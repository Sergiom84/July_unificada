# Checklist: Entrega de web de cliente

## Antes de construir

- [ ] Objetivo de la web claro
- [ ] Cliente y sector definidos
- [ ] Dominio decidido
- [ ] Servicios principales definidos
- [ ] Telefono, email, direccion y redes confirmados
- [ ] Imagenes disponibles o plan para generarlas/buscarlas
- [ ] Competidores o referencias revisadas
- [ ] Stack elegido

## Contenido

- [ ] Hero claro
- [ ] Servicios completos
- [ ] CTA principal
- [ ] CTA secundario por WhatsApp o email
- [ ] Contacto visible
- [ ] Paginas legales
- [ ] Textos revisados
- [ ] No quedan textos genericos de plantilla

## SEO

- [ ] Title unico por pagina
- [ ] Description unica por pagina
- [ ] Canonical correcto
- [ ] Open Graph
- [ ] Imagen OG real
- [ ] Schema JSON-LD si aplica
- [ ] Sitemap generado
- [ ] Robots correcto
- [ ] Alt text en imagenes
- [ ] Paginas por ciudad o servicio si aporta valor

## Formularios

- [ ] Validacion frontend
- [ ] Envio real probado
- [ ] Mensaje de exito
- [ ] Mensaje de error
- [ ] Email o almacenamiento confirmado
- [ ] Checkbox de privacidad
- [ ] Prueba en entorno de deploy

## Responsive y UX

- [ ] Mobile
- [ ] Tablet
- [ ] Desktop
- [ ] Menu movil
- [ ] CTAs visibles
- [ ] Textos no se solapan
- [ ] Imagenes no se deforman
- [ ] Velocidad razonable

## Deploy

- [ ] `npm run build` pasa
- [ ] Configuracion de deploy en repo
- [ ] Variables de entorno configuradas
- [ ] Dominio conectado
- [ ] HTTPS activo
- [ ] Redirecciones revisadas
- [ ] Cache headers revisados

## Si usa Supabase

- [ ] Usar pooler en produccion
- [ ] RLS activado donde corresponda
- [ ] No exponer `service_role`
- [ ] Variables separadas frontend/backend
- [ ] Probar conexion en deploy
- [ ] Probar consultas reales

## Cierre

- [ ] Checklist final guardada en Notion
- [ ] Problemas y soluciones documentados
- [ ] Repo enlazado desde Notion
- [ ] Decisiones tecnicas importantes registradas
- [ ] Cliente tiene instrucciones basicas
